# Leadership Memory 分析器

## 任务

从原材料中提取这位领导 **{name}** 的“领导记忆”，用于构建 Leadership Memory。

只写有依据的内容，没有证据就标注“原材料不足”。

## 提取维度

### 1. 身份与职责
- 所在组织/条线/层级
- 分管范围
- 关注的核心目标
- 与用户的汇报关系

### 2. 常抓重点
- 领导反复强调的关键词
- 最关注的业务目标/指标
- 对创新、落地、试点、规模化的态度
- 对协同、合规、安全、预算的关注点

### 3. 风险底线
- 哪些事情不能碰
- 哪些表达方式容易被否
- 哪些场景一定要求证据、数据、边界说明
- 是否偏好“先试点后推广”

### 4. 典型批示与偏好
- 喜欢的汇报结构
- 常见批示语气
- 典型表态方式：支持 / 暂缓 / 补充 / 否决
- 喜欢什么样的材料：一页纸、三点式、问题导向、表格化等

### 5. 典型案例
- 通过过哪些事，理由是什么
- 否掉过哪些事，理由是什么
- 对资源、周期、责任分工的典型要求

## 输出要求

- 语言：中文
- 优先引用原话或原始表述
- 避免心理分析，聚焦管理行为和决策证据
- 输出结构可直接用于生成 `memory.md`
