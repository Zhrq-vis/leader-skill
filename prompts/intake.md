# 基础信息录入脚本

## 开场白

```text
我来帮你创建一个 领导.skill。

这不是简单模仿口吻，而是蒸馏一位领导的：
- 决策逻辑
- 汇报偏好
- 风险边界
- 会场追问方式

我会先问你 3 个问题，再选择性导入讲话、纪要、批注、邮件或口述材料。
```

## 问题序列

### Q1：领导代号/姓名（必填）

```text
先给这位领导一个代号吧。
可以是真名、岗位称呼、匿名代号都行。

比如：王总 / 张主任 / 研发一把手 / 条线领导
```

### Q2：岗位背景（可跳过）

```text
用一句话描述这位领导的岗位背景。

可以包括：
- 所处分管条线
- 层级 / 岗位
- 你与他的关系（是否直接汇报）
- 典型负责事项

示例：
“央企研发条线负责人，分管科研试点和成果转化，我需要向他汇报项目。”
```

### Q3：风格画像（可跳过）

```text
再用一句话描述他的领导风格。

可以包括：
- 喜欢什么样的汇报
- 讨厌什么样的表达
- 重结果 / 重风险 / 重程序 / 重政治站位 / 重协同
- 会不会当场拍板，还是喜欢让你先补材料

示例：
“结果导向，风险敏感，讨厌空话，喜欢先讲结论后讲依据。”
```

## 汇总确认

```text
好的，我整理一下：

  代号：{name}
  岗位背景：{summary}
  风格画像：{style_summary}

这样对吗？确认后进入下一步（导入原材料）。
```
