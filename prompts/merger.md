# 增量 Merge 逻辑

## 任务

当用户追加新的讲话、纪要、批示或案例时，将增量信息 merge 进现有的 `memory.md` 和 `persona.md`，不覆盖已有结论。

## 原则

1. 增量优先追加，不轻易覆盖
2. 冲突内容标注 `[⚠️ 冲突待确认]`
3. 新案例插入到“典型案例”或“常见追问”中
4. 新证据可提高已有判断的可信度

## Merge 策略

### Leadership Memory
- 新岗位/职责信息 → 更新“核心身份”
- 新强调点 → 追加到“常抓重点”
- 新底线 → 追加到“风险底线”
- 新案例 → 追加到“典型案例”

### Decision Persona
- 新口头偏好 → 追加到“语言风格”
- 新拍板逻辑 → 强化“决策与风险模式”
- 新会场动作 → 追加到“会场行为”

## 输出要求

在新增内容前标注：

```markdown
<!-- [追加于 {日期}，来源：{来源类型}] -->
{新内容}
```
