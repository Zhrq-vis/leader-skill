# Decision Persona 分析器

## 任务

从原材料中提取这位领导 **{name}** 的决策人格，用于驱动对话中的“怎么看、怎么问、怎么拍板”。

## 提取维度

### 1. 语言风格
- 常见句式：先讲结论 / 三点说明 / 先问问题 / 先压风险
- 口头偏好：比如“先别急”“讲重点”“落到动作上”
- 正式度：偏口语 / 偏公文 / 会场压迫型 / 平稳克制型
- 表达长度：短促拍板型 / 展开分析型

### 2. 会场行为
- 先听完再问，还是中途打断
- 喜欢追问数据、边界、责任，还是追问战略意义
- 会不会明确给动作项
- 是当场拍板还是回去补材料

### 3. 决策模式
- 结果导向 / 程序导向 / 风险导向 / 平衡导向
- 倾向快决策还是稳妥推进
- 对试点、创新、预算、协同的态度
- 面对不确定性时的默认策略

### 4. 风险偏好
- 风险厌恶 / 审慎接受 / 愿意冒险
- 关注哪些风险：政治、合规、安全、资源、舆情、落地、群众感知
- 如何要求兜底机制

### 5. 对下属汇报的期待
- 喜欢先讲结论还是先讲背景
- 是否接受技术细节，接受到什么程度
- 喜欢数据、图表、案例还是口头概括
- 最不喜欢什么：空话、堆概念、没有边界、没有对比、没有下一步动作

## 输出格式

输出一个 5 层 Persona 结构草稿，每层是具体规则，不是抽象标签。
